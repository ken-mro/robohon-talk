## 概要 / Overview

ロボホンアプリ開発用ツール（以下「本ツール」）は、ロボホンアプリの開発・デバッグ時に役立つ機能を提供する公式ツール群です。主に **2 つの機能** を提供します:

1. **発話のイントネーション調整** — 発話テキストのカナ表記・アクセント・音の高さ/長さ・感情(emotion)・ポーズ/休止を GUI で調整し、調整結果を HVML の `<action>` タグとして書き出す。
2. **ロボホンの発話および、ユーザの発話の遠隔エミュレーション** — ブラウザ経由でロボホンに任意の発話をさせたり、ユーザ発話を擬似的に聞かせたり (LVCSR エミュレーション) して、開発中アプリの挙動をテストする。

本ツールは次の 2 コンポーネントで構成されます:

| コンポーネント | 種別 | 役割 |
|---|---|---|
| **RbDevTool_Vxx_xx_xx.apk** (例: `RbDevtool_V01_00_00.apk`) | ロボホン端末上の Android アプリ「開発ツール」 | サーバ機能を提供。トグル ON で受付アドレス (`http://xx.xx.xx.xx:48000`) を公開し、PC ツールやブラウザからの発話/listen 要求を受け付ける。 |
| **KanaHighLow_Vxx_xx_xx.exe** | Windows PC 用アプリ「イントネーション調整ツール」 | カナ/アクセント/感情を GUI 編集し、調整した発話を端末に送って試聴し、HVML を出力する。 |

> **重要 — 前提バージョン:** 本ツールの利用にはロボホンの **ビルド番号 03.01.00 以降** が必要です。
> 「設定 - 端末情報 - ビルド番号」を確認し、対象バージョンでない場合は「設定 - 端末情報 - ソフトウェア更新」よりアップデートしてください。
> 利用には別紙「ロボホンアプリ開発用ツール」利用規約への同意が必要です。

## 2. ロボホンアプリ開発用ツールの概要 / Tool Overview & Installation

本ツールが提供する機能:

- ① 発話のイントネーション調整
- ② ロボホンの発話および、ユーザの発話の遠隔エミュレーション

前提条件（再掲）: ロボホンのビルド番号 **03.01.00 以降**。利用規約への同意が必要。

### 2.1 ツールのインストール / Installation

#### A. 端末側アプリ (`RbDevTool_Vxx_xx_xx.apk`)

ロボホン端末にインストールする「開発ツール」アプリ。SDK に同梱 (例: `RbDevtool_V01_00_00.apk`)。手順:

1. **USB デバッグの設定をする。** 詳細は参考資料 [1] (スタートガイド) を参照。
2. **USB ケーブルを接続する。**
3. コマンドプロンプトにて **`adb install`** コマンドで apk をインストールする。
   ```
   adb install RbDevtool_V01_00_00.apk
   ```

#### B. PC 用イントネーション調整ツール (`KanaHighLow_Vxx_xx_xx.exe`)

- **インストール不要。** 利用時に実行ファイル (exe) を直接実行する。
- **アンインストール:** 実行ファイルと自動生成されるファイルを削除するだけ。**レジストリは使用しない。**
- **必須環境:** **Microsoft Office IME** が必要 (カナ変換に使用)。
- **動作確認 OS:** Windows 7 / Windows 8.1 / Windows 10。

## 3. 各機能の利用方法 / Using Each Feature

### 3.1 発話のイントネーション調整 / Speech Intonation Adjustment

> **日本語のみサポート。** 発話テキストのカナ・アクセント・音高/音長・感情・ポーズ/休止を GUI で調整し、HVML を出力する機能。

#### 利用手順 / Steps

1. **接続の準備:**
   - **Wi-Fi 経由の場合:** ロボホンを任意の Wi-Fi アクセスポイントに接続する。
   - **USB 接続の場合:** USB デバッグを設定し、ロボホンを USB ケーブルで PC につなぐ。
   - 詳細は参考資料 [1] 参照。

2. **端末側ツールを起動:** ロボホンのホーム画面より **「開発ツール」** を起動する。
   - Wi-Fi 接続されていない時は「接続しているネットワーク」が **未接続** と表示される → 改めて Wi-Fi 接続設定を確認する。

   **［図 3-1 開発ツール画面］** — 端末アプリ「開発ツール」のメイン画面。画面構成:
   - タイトル: **開発ツール** （右上に ⓘ インフォアイコン）
   - **ON / OFF トグルスイッチ** （初期状態 OFF）
   - **受付アドレス**: `http://192.168.10.159:48000` のような URL を表示（ポートは **48000**）
   - **接続しているネットワーク**: 接続中の Wi-Fi AP (SSID) を表示
   - 下部に 戻る(←) / ホーム(⌂) ナビゲーションボタン

3. **トグルスイッチを「ON」にする。** （これでサーバ機能が起動し、受付アドレスが有効になる）

4. **PC を同一 AP に接続:** ロボホンが接続している Wi-Fi AP（「接続しているネットワーク」に表示された AP）と **同一の AP に PC を接続** する。
   - ※ **USB 接続でのご利用の際は不要。**

5. **PC 用ツールを起動:** **`KanaHighLow_Vxx_xx_xx.exe`** を起動する。

   **［図 3-2 イントネーション調整ツール画面］** — `KanaHighLow - 無題.txt` というタイトルの Windows GUI アプリ。構成:

   - **メニューバー:** `ファイル` / `ポーズ` / `感情` / `休止` / `棒読み`
   - **発話テキスト入力エリア** （大きな複数行テキストボックス）— 発話させたい全文を入力。
   - **behavior id** （右上のコンボボックス）— 発話時に再生する動作(behavior)の ID を入力/選択。
   - **接続 (経路選択):** ラジオボタン **`USB`** / **`Wi-Fi`** と、Wi-Fi 用 IP アドレス入力欄 (例: `192.168.11.3`)。
   - **発話ボタン:** **`発話(全文)`** / **`発話(選択)`**
   - **`←<undo>` / `<redo>→`** ボタン
   - **入力(カナ)** テキストボックス — 選択範囲のカナ表記を表示/編集。
   - **品詞コンボボックス** （例: `普通名詞`）— カナの品詞を指定（アクセント推定に使用）。
   - **イントネーション調整 GUI** （中央エリア）— 文字が入った「箱」が音節ごとに並ぶ。箱の高さ＝音の高さ、長さ＝音の長さ。
   - **結果** テキストボックス（下部）— 生成された `<vtml_phoneme …>…</vtml_phoneme>` 等の文字列を表示。
   - **`発話(単語)`** / **`挿入`** ボタン（下部右）。

6. **経路を選択:** 「接続」から利用する経路 (USB / Wi-Fi) を選択する。
   - **Wi-Fi 接続の場合:** 図 3-1 画面に表示されている **IP アドレス部 (`192.xxx.xxx.xxx`)** の値を入力する。

7. **発話させる:** 発話内容と **behavior ID** を入力し、
   - **`発話(全文)`** を押下 → 入力テキスト全文を発話。
   - **`発話(選択)`** を押下 → 入力テキストの一部を範囲選択しておくと、その **部分のみを発話** できる。

8. **イントネーションを調整する:** 入力テキストのうち、イントネーションを調整したい部分を **範囲選択** すると、カナ表記とイントネーション調整用 GUI が表示される。
   - 文字が入った **箱の高さ・長さを変更** → 音の高さ・長さを変更。
   - **箱をクリック** → アクセントの位置や区切り位置を調整。
   - **`発話(単語)`** ボタン → 調整した部分のみを発話して試聴。
   - **`挿入`** ボタン → 調整済みテキストを全文へ挿入（反映）。

   **［図 3-3 イントネーション調整ツール画面 2］** — 例: 全文「おはよう、こんにちは、さようなら」のうち「こんにちは」を選択。
   - 入力(カナ) = `コンニチワ`、品詞 = `普通名詞`、behavior id = `assign`。
   - 調整 GUI に `コ / ン / ニ / チ / ワ` の箱が並び、高さ・位置を調整。
   - 結果欄: `<vtml_phoneme alphabet="x-pentax"; ph="コ6^ン3ニ/チ3ワ79[普通名詞]">こんにちは</vtml_phoneme>`

9. **感情 (emotion) を調整する:** 入力テキストのうち emotion を付けたい部分を範囲選択し、メニュー **「感情」** から感情を選択すると `vtml_emotion` (emotion タグ) が挿入される。
   - **「感情」メニューの選択肢:** `happiness1` / `happiness2` / `anger1` / `anger2` / `sadness1` / `sadness2`
     （末尾の数字は感情の level に対応: 例 `happiness2` → `category="happiness" level="2"`）
   - その他、メニューから **ポーズタグ (`ポーズ`)** や **休止タグ (`休止`)**、**棒読み (`棒読み`)** も追加できる。

   **［図 3-4 イントネーション調整ツール画面 3］** — 「さようなら」を選択し「感情」メニューを開いた状態。
   - 入力(カナ) = `サヨウナラ`、調整 GUI に `サ / ヨ / ウ / ナ / ラ` の箱。
   - 結果欄: `<vtml_phoneme alphabet="x-pentax"; ph="サヨウナラ[普通名詞]">さようなら</vtml_phoneme>`

10. **HVML を書き出す:** イントネーション調整した結果は、**`発話(全文)`** ボタンを押下することで、本ツールと同フォルダ内に
    **`jp_co_sharp_android_rb_devtool_intonation.hvml`** が出力される。
    このファイルから **`<action>` タグ部をコピーして、実際に利用する HVML ファイルに貼り付ける。**
    - ※ HVML 形式で記述する場合、**禁則文字のエスケープが必要** （`<` → `&lt;`、`>` → `&gt;`、`"` → `&quot;` 等）。本ツールはエスケープ済みの形で出力する。

#### 出力 HVML の抜粋 / Example output (verbatim)

`jp_co_sharp_android_rb_devtool_intonation.hvml` より抜粋:

```xml
~中略~
    <topic id="accost" listen="false">
       <action index="1">
          <speech>おはよう、&lt;vtml_phoneme alphabet=&quot;x-pentax&quot;; ph=&quot;コ6^ン3ニ/チ3ワ79^[普通名詞]&quot;&gt;こんにちは&lt;/vtml_phoneme&gt;、&lt;vtml_emotion category=&quot;happiness&quot; level=&quot;2&quot;&gt;さようなら&lt;/vtml_emotion&gt;
          </speech>
          <behavior id="assign" type="normal"/>
       </action>
    </topic>
  </body>
</hvml>
```

- アンエスケープした実体は概ね次の通り (HVML リファレンス [3] の `<speech>` 仕様に従う):
  ```
  おはよう、<vtml_phoneme alphabet="x-pentax"; ph="コ6^ン3ニ/チ3ワ79^[普通名詞]">こんにちは</vtml_phoneme>、<vtml_emotion category="happiness" level="2">さようなら</vtml_emotion>
  ```
- `<behavior id="assign" type="normal"/>` のように、調整時に指定した behavior id が `<action>` 内に書き出される。

### 3.2 ロボホンの発話および、ユーザの発話の遠隔エミュレーション / Remote Emulation of RoBoHoN Speech & User Speech

> ブラウザ経由で、(1) ロボホンに任意の発話をさせる、(2) ユーザ発話を擬似的に聞かせる (LVCSR エミュレーション) ことができる機能。アプリのトリガ動作テストに使う。

#### 利用手順 / Steps

1. **3.1 の手順 1〜4 まで進める** （Wi-Fi/USB 準備 → 「開発ツール」起動 → トグル ON → PC/端末同一 AP 接続。ブラウザ利用のため PC 用 exe の起動は不要）。

2. **受付アドレスにブラウザでアクセス:** 「開発ツール」画面の **「受付アドレス」** に表示される
   **`http://xx.xx.xx.xx:48000`** （ポート **48000**）を、**スマホや PC のブラウザのアドレスバーに入力して接続** する。

   **［図 3-5 遠隔操作画面］** — タイトル **「RoBoHoN DevTool」** の Web ページ。2 つのセクションで構成:

   - **ロボホンに発話をさせる** (Make RoBoHoN speak):
     - 発話テキスト入力欄（例: `おはよう`）
     - behavior id 入力欄（例: `assign`）
     - **`speech`** ボタン → 入力テキストをロボホンに発話させる（指定 behavior id で動作）。
   - **ロボホンに話しかける** (Talk to RoBoHoN / ユーザ発話エミュレーション):
     - **`Lvcsr:Basic`** 入力欄（例: `おはよう`）— 認識結果テキスト（表記）。
     - **`Lvcsr:Kana`** 入力欄（例: `オハヨウ`）— 認識結果のカナ表記。
     - **`listen`** ボタン → ロボホンがそのユーザ発話を聞いたものとして擬似的に発生させる。

3. **発話/listen させる:** 図 3-5 の画面で、各テキストボックスに入力して
   - **`speech`** ボタン → ロボホンに発話させる。
   - **`listen`** ボタン → ロボホンがユーザ発話を聞いた時のエミュレーションを行う。

#### 補足 / Notes

- **ユーザ発話のエミュレーション** は、参考資料 **[4] アプリ開発ガイドライン**「**5.14.1 ユーザ発話を擬似的に発生させる**」の機能と **同等の振る舞い** となる。
  - `listen` の `Lvcsr:Basic` / `Lvcsr:Kana` は、音声認識 (LVCSR) の認識結果（表記＋カナ）をそのまま流し込むイメージ。実機にマイクで話しかけずに、開発中アプリの発話受付 (`listen`) ロジックを検証できる。
- `speech` は、実機で実際にロボホンを喋らせ、behavior（モーション）も同時にテストできる。
